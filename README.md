**Requirements**
- A web browser to load the frontend interface  
- Node.js installed on the system  
- Basic understanding of HTML, CSS, and JavaScript  
- Knowledge of handling JSON data  
- Ability to run a local Node.js HTTP server  

**Technologies Used**
- **HTML** for structuring the Todo app interface  
- **CSS** for basic styling  
- **Vanilla JavaScript** for DOM manipulation and client-side logic  
- **Fetch API** for sending POST and GET requests  
- **Node.js (HTTP module)** to create a server without Express  
- **File System (fs)** to read and write todos to `data.json`  
- **Path module** for handling file paths  
- **JSON** for storing and retrieving todo data  

**Features Used**
- DOM selection and event listeners  
- Dynamic element creation for displaying tasks  
- Basic input validation on the frontend  
- Handling POST requests to save tasks  
- Serving static files (HTML/CSS/JS/JSON) manually  
- Reading and writing JSON files on the server  
- Routing based on request method and URL
